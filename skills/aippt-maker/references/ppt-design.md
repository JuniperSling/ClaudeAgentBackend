# PPT 内容设计规范

规划大纲、填写 `content_spec` 和 `theme` 字段时参考本文件。生成 HTML 时从 `research.md` 提炼内容也需遵循本文件中的内容组织原则。

---

## 内容组织原则

从 `research.md` 素材转化为幻灯片内容时，MUST 遵循以下四项原则：
1. 核心观点提取：将长文本转化为精炼的短句，确保每页只有一个核心思想。每页的标题应能独立概括该页的关键结论，观众即使只读标题也能把握演示脉络。
2. 结构化转换：优先使用列表、表格或图表替代大段落纯文本描述。当内容包含对比关系时用表格，包含流程/步骤时用时间轴或编号列表，包含层级关系时用分栏卡片。避免任何一页出现超过三行的连续段落文字。
3. 结论导向：采用金字塔原理，结论先行，论据支撑。每页先呈现核心结论或观点，再用数据、案例、对比等作为支撑。观众应在前 3 秒内抓住该页要传达的信息。

## 页面类型

| 类型 | 用途 | 特点 |
|------|------|------|
| `title` | 封面页、结尾页 | 大标题居中，视觉冲击力强 |
| `section` | 章节过渡页 | 章节标题 + 简短说明，承上启下（慎用，见下方规则） |
| `content` | 内容页 | 核心信息展示，有结构有层次 |

### section 页使用规则

section 页的本质是**视觉呼吸点**——让观众知道"我们进入了一个新话题"。但过多的呼吸点会让节奏碎片化，内容显得单薄。

**何时使用 section 页：**
- 话题发生**本质切换**（如从"问题分析"转到"解决方案"），而非同一话题的递进
- 演示文稿总页数 ≥ 10 页，且内容可自然分为 2–4 个独立章节
- 章节之间的内容页数量 ≥ 3 页（每个章节至少有 3 页实质内容才值得用 section 页分隔）

**数量上限：** section 页数量不超过 `总页数 × 20%`（向下取整），且最多 4 页。例如 15 页的演示文稿最多 3 页 section，8 页的演示文稿最多 1 页 section。

## theme 字段规范

theme 是自然语言风格描述，包含以下 **5 个必填维度**：

1. **整体风格**：用一句话概括视觉调性，可附带英文名称或类比参照物（如"类似GitHub视觉体验"）。关键词粒度：风格流派 + 情绪基调（如"极简""波普艺术""商务专业"）。
2. **背景**：说明背景色/底色。需包含具体色值（hex 格式，至少给出主背景色）；可附加质感描述（如"纯色""网格""金属质感"）；明确是浅色系还是深色系。
3. **字体**：分角色指定字体风格——标题字体、正文字体各一；如有特殊场景字体（代码/命令等宽字体）单独说明；中英文字体需分别指定。
4. **配色**：给出完整调色板——主色调（+ hex）、辅色调（+ hex）、强调色/点缀色（+ hex）；当背景为深色时需额外指定文字色。
5. **视觉元素**：描述装饰语言和设计手法——图形语言（如扁平化几何、线稿图标）、装饰元素（如细线、圆点、代码块样式）、排版特征（如留白充足、表格流程图）。

### 示例

```
1. 整体风格：Github极简风格，代码风格元素，简洁线条，技术感。\n2. 背景：深色背景 #0d1117 或 #24292e，纯色或极简网格，深色系。\n3. 字体：标题使用等宽字体风格（如 Courier New, Consolas），正文使用简洁现代无衬线体（如 Helvetica, Arial）。\n4. 配色：主色调深灰/黑 #0d1117，辅色浅灰/白 #c9d1d9（文字色），强调色 Github 绿 #238636 或蓝 #58a6ff。\n5. 视觉元素：代码块样式，终端窗口样式，极简线条，留白充足，技术感图标装饰。
```

```
1. 整体风格：简约技术风格，黑白线稿风格，简洁专业。\n2. 背景：纯白 #FFFFFF 或极浅灰 #F3F4EF，浅色系。\n3. 字体：现代无衬线体（如 Arial, Helvetica）。\n4. 配色：主色调黑/深灰 #333333，辅色调浅灰 #999999，点缀色极简线条色。\n5. 视觉元素：黑白线稿图标，极简几何图形，清晰的表格和流程图。
```

## content_spec 字段规范

content_spec 描述这一页**要呈现哪些具体要点**——要点本身要列出来，但措辞展开和数据细节由执行方从 `research.md` 中提炼。把它理解为"告诉执行方这页要讲什么、分哪几块"，而不是把每个字都写死。

布局结构不需要在 content_spec 中指定，由生成 HTML 时根据内容量自主判断。

空泛描述（如"介绍产品"、"展示数据"）是错误的——需要说清楚具体是哪些要点、哪几个维度，执行方才能从 research.md 中准确提炼对应内容。

### 三种页面类型的 content_spec 示例

**type: title（封面页 / 结尾页）**
```
封面：主标题"Agent Harness Engineering"，副标题"构建可靠 AI Agent 的运行时控制框架工程学"，底部标注"2026 · AI Engineering Series"。
```

**type: section（章节过渡页）**
```
章节过渡：引出"什么是 Harness Engineering"，点出两个核心论点：① Harness 是 Agent 系统的运行时控制层；② 模型能力固定时，Harness 决定系统表现上限。
```

**type: content（内容页）**
```
对比 Prompt Engineering 与 Harness Engineering 的四项核心差异：静态字符串 vs 动态上下文、单次调用 vs 长时运行、无状态 vs 状态持久化、人工干预 vs 自主执行+人工掌舵。
```

```
展示2024年三大核心业务板块的表现：电商（营收占比、同比增速）、广告（营收占比、同比增速）、云计算（营收占比、同比增速），并说明增长最快的板块及原因。
```

```
介绍产品差异化竞争优势：列出与竞品相比的五个核心差异点，每项说明具体体现在哪里。
```
