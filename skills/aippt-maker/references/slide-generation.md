# 逐页 HTML 生成指南

本文件描述「批量生成 PPT」第 6 步的具体执行方式，根据当前环境选择并行或串行模式。

生成每页 HTML 时，内容来源有两个：
- **`research.md`**：项目目录根的内容素材文件，包含所有具体文案、数据、论据
- **`content_spec`**：该页的内容概要，指明这页聚焦的方向/角度

执行方需要从 `research.md` 中找出与 `content_spec` 方向匹配的内容，按照 `ppt-design.md` 中「内容组织原则」（核心观点提取、结构化转换、结论导向）进行内容转化，再结合 `html-spec.md` 的技术规范渲染成 HTML。布局结构根据内容量自主判断。

---

## 判断执行模式

| 条件 | 执行模式 |
|------|----------|
| 明确知道当前平台的子 agent 支持文件写入（如 Claude Code） | **并行模式**：同时启动多个子 agent，每个负责一页 |
| 其他所有情况（包括不确定写入权限的情况） | **串行模式**：逐页顺序生成，自行生成每页 HTML |

**默认走串行**。子 agent 能否写入文件往往要试了才知道，不同平台行为不一致，不确定时统一走串行，避免子 agent 生成内容却无法落盘导致的失败和重试浪费。

---

## 并行模式（子 agent）

**优点**：每页上下文完全隔离，速度快，不同页面的内容和样式不会互相污染。

为每页创建一个子 agent，使用以下 prompt 模板（填入对应字段后发送）：

```
请严格按照 <SKILL_DIR>/references/html-spec.md 中的技术规范，为以下幻灯片生成一个完整的 HTML 文件。

【内容素材】
读取以下路径的 research.md，从中提炼与本页 content_spec 方向相关的内容：
- research.md 路径：<项目目录>/research.md

【幻灯片信息】
- 输出文件路径：<项目目录>/slides/<file>
- theme：<theme>
- type：<type>
- title：<title>
- content_spec：<content_spec>

content_spec 描述了这一页的内容方向/角度，具体文案和数据请从 research.md 中提炼。从素材到幻灯片内容的转化须遵循以下原则：每页只保留一个核心思想，长文本提炼为精炼短句；优先用列表/表格/图表替代大段文字；结论先行、论据支撑（金字塔原理）。如果 research.md 中有直接相关的内容，优先使用原文。布局结构请根据内容量自主判断，选择最适合的排版方式（分栏/网格/时间轴等），确保页面充实、不空洞。
直接输出完整 HTML（从 <!DOCTYPE html> 开始），写入上述路径，不要输出任何解释文字。
```

所有子 agent 同时启动，等待全部完成后再进行下一步。

---

## 串行模式（无子 agent）

**注意**：串行模式下你直接承担每页 HTML 的生成工作。`research.md` 内容已在当前上下文中（你在生成大纲前已读取），无需再次显式读取。只需在生成 HTML 前读取一次 `<SKILL_DIR>/references/html-spec.md`（整个流程读一次即可）。

逐页按以下步骤执行：

1. 从 `presentation.json` 读取当前页的 `theme`、`type`、`title`、`content_spec`
2. 根据 `content_spec` 的方向，从已有的 `research.md` 内容中提炼出该页所需的具体文案和数据
3. 严格按照 html-spec.md 的技术规范生成完整 HTML，布局根据内容量自主判断
4. 将 HTML 写入 `<项目目录>/slides/<file>`
5. 继续下一页

**关键原则**：`research.md` 是内容的权威来源，优先使用其中的原始内容；布局要适配内容量，页面应充实有设计感，不要只是把文字堆在白底上。

---

## 通用注意事项

无论哪种模式，每页生成后都应简单检查文件是否成功写入（文件存在且非空）。若某页生成失败，单独重试该页即可，不影响其他页面。
